## 1. スキル負債（最重要）

### 1.1 🔴✅ 唯一のスキルが「無効な形式」で読み込まれない
- **事象:** リポジトリ唯一の資産 `ai-agent-architect.md` は**リポジトリ直下のフラットな Markdown**であり、Agent Skill として認識されない。
- **根拠:** スキルは「`<name>/SKILL.md`（YAML frontmatter に `name` と `description`）」というディレクトリ構成が必須（Anthropic 公式仕様）。フラットな `*.md` は Claude Code にも claude.ai スキルとしても登録されない。実際、本アカウントの有効スキル一覧（11件）にも本ファイルは存在せず、**完全に孤立（dead）**していた。
- **付随:** 本文冒頭に claude.ai スキルビルダー UI の定型文（「スキル名:／説明文:／アイコン案:／スターター3選」）がそのまま貼られており、スキル指示ではなくメタ情報だった。frontmatter が無いため**起動条件（trigger）も定義されていなかった**。
- **対応（本PR）:** `ai-agent-architect/SKILL.md` へ変換。`name`/`description`（何を・いつ使うか）を付与、UI定型文を除去、`references/` に出典を分離。旧フラットファイルは削除。

### 1.2 🟠 重複ブランチ
- **事象:** `claude/code-debt-inventory-plk811` と `claude/ai-agent-knowledge-base-5kn6t` が**同一内容**（diff 空）。単一の正がない。
- **推奨:** 一方に集約し、もう一方はマージ後に削除。

### 1.3 🟠✅ コンテンツ正確性（出典なしの断定数値）
- **事象:** 知識ベースが具体的数値を事実として断定（例: 「67%」「81%向上」「20%短縮」）。一部（McKinsey「Lilli」侵害: 約2時間・約4,650万メッセージ・システムプロンプト露出）は**実在の2026年3月事案として裏が取れた**が、他の百分率は出所が揺れる。
- **リスク:** 「専門家」を名乗るスキルが未検証の数値を伝播させ、意思決定を誤らせる。
- **対応（本PR）:** 裏が取れた事例には出典を付与。裏取り困難な数値は「解説用の目安（要一次検証）」と明記し、`references/sources.md` に根拠の強さ（🟢🟡⚪）付きで整理。

## 2. メモリ負債（CLAUDE.md / ドキュメント）

### 2.1 🟠✅ リポジトリに説明・規約が皆無
- **事象:** `README.md` も `CLAUDE.md` も無し。`skills` という名前のリポジトリなのに、スキルの置き方・命名規約・構成の記述がゼロ。新規セッションやコントリビュータが構造を再発見するコストが発生。
- **対応（本PR）:** `README.md` を追加。ディレクトリ構成・`SKILL.md` frontmatter 規約（`name` の予約語制限含む）・収録スキル一覧・メンテ方針を記載。実質的に本リポジトリのメモリとして機能。

> 注: ユーザー/グローバルの `~/.claude/CLAUDE.md` は存在せず（負債ではない）。ハーネス管理の hook（`session-start-*`, `stop-hook-*`）は正規のもので棚卸し対象外。

## 3. プラグイン負債

### 3.1 🟡 プラグイン 0件
- **事象:** 有効プラグインなし（`ListPlugins` → 空）。それ自体は負債ではない。
- **推奨（任意）:** 将来スキルが増えるなら、marketplace 形式のプラグインとして束ねると配布・バージョン管理が容易。現時点では過剰。

## 4. コネクタ（MCP）負債

有効化済みコネクタ 9件のうち **3件が休眠**。スキル自身の「最小権限」原則にも反する。

| コネクタ | 状態 | 判定 |
| --- | --- | --- |
| Canva / Figma / Gmail / Google Drive / Notion / Slack | connected・チャット有効 | ✅ 稼働中 |
| **Google Calendar** | installState=unknown・チャット無効 | 🟠 休眠 |
| **Linear** | installState=unknown・チャット無効 | 🟠 休眠 |
| **Vercel** | installState=unknown・チャット無効 | 🟠 休眠 |

- **リスク:** 未認証/未使用の接続は攻撃面と混乱の元。
- **推奨:** 使うなら認証してチャットで有効化、使わないなら削除。加えて、再現性が要るなら利用コネクタをリポジトリの `.mcp.json` に固定化（現在は全てアカウント単位でリポジトリに紐づかない）。

## 5. 設定 / ハーネス負債

### 5.1 🟡 ユーザー `settings.json` 不在（権限許可リスト未整備）
- **事象:** `~/.claude/settings.json` も プロジェクト `.claude/settings.json` も無し。権限は既定のまま。
- **推奨（任意）:** よく使う読み取り専用コマンドを許可リスト化すると権限プロンプトが減る（`/fewer-permission-prompts`）。小規模な現状では必須ではない。
