# Worker スキルテンプレート（実行者: Sonnet 5 サブエージェント）

手順が支配的な業務用。Sonnet 5 は行間を埋める力が Opus より弱いため、
**曖昧さを意図的に潰す**: 入出力を契約として宣言し、手順を完全列挙し、機械的な自己検証で締める。

サブエージェントは**コールドスタート**（会話文脈を引き継がない）。
スキル本文と呼び出しプロンプトだけで作業が完結するよう、前提をすべてスキル内に書く。

```markdown
---
name: {{skill-name}}
description: {{何をするスキルか}}。{{どんな依頼・状況のときに使用するか}}。
---

# {{Skill Title}}

> 想定実行者: Sonnet 5 サブエージェント（Worker プロファイル / コールドスタート前提）

## 役割（Role）
{{担う作業者としての立場。判断はしない・改変しない範囲を明記}}

## 入力契約（Inputs）
呼び出し側（Manager）がプロンプトで渡すべき値。欠けていたら作業せず不足を報告する。
- {{入力1: 型・形式・例}}
- {{入力2: 型・形式・例}}

## 出力契約（Outputs）
- {{返す成果物の厳密な形式（フォーマット・文字数上限・ファイルパス等）}}
- 結論だけを返す。経過説明・謝辞・提案を含めない

## 前提（Context）
コールドスタートでも迷わないための固定情報。
- {{対象ディレクトリ・命名規約・参照ファイルの絶対条件}}

## 制約（Constraints）
- {{してよい操作の範囲。肯定文で}}
- 手順にない判断が必要になったら、作業を止めて判断要求として報告する（勝手に補完しない）

## 手順（Sequence）
1. {{完全列挙。各ステップに具体コマンド・対象・成功条件}}
2. {{条件分岐は「もし◯◯なら△△」の形で明示}}

## 自己検証（Self-Verification）
返答前に必ず実行し、結果を出力に含める。
- {{機械的チェック（例: `python3 skills/skill-foundry/scripts/validate_skill.py <dir>` が PASS）}}
- {{検証が FAIL の場合: 修正を1回試行し、なお FAIL なら失敗として報告する}}

## 完了条件（Completion Criteria）
{{出力契約を満たし、自己検証が PASS したら終了。それ以外は失敗報告で終了}}

## 出典
[`references/sources.md`](references/sources.md) を参照。
```

## 記入時の注意
- 「入力契約」は Manager が呼び出しプロンプトを組むときの仕様書になる。曖昧な項目を残さない
- 手順の各ステップは「Sonnet 5 が裁量なしで実行できるか」で自己点検する。できないステップは Manager 側の判断に戻す
- 自己検証は必ず機械的チェック（コマンド・スクリプト）を1つ以上含める。自己評価文（「問題ありません」等）は検証と見なさない
